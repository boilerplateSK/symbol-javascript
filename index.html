🔑 JavaScript Symbol Data Type

What is a Symbol?

Symbol is a primitive data type introduced in ES6 (2015). Every Symbol is unique and immutable, making it perfect for creating unique identifiers that won't conflict with other properties.

🔧 Basic Symbol Creation

Creating Symbols
// Creating symbols const symbol1 = Symbol(); const symbol2 = Symbol(); const symbol3 = Symbol('description'); const symbol4 = Symbol('description'); console.log(symbol1); // Symbol() console.log(symbol3); // Symbol(description) console.log(symbol1 === symbol2); // false (unique!) console.log(symbol3 === symbol4); // false (still unique!) console.log(typeof symbol1); // "symbol"
🎯 Key Point: Every Symbol is unique, even if they have the same description!

🎯 Real-World Use Cases

1. Private Object Properties

Private Properties with Symbols
const _privateData = Symbol('private'); class User { constructor(name, age) { this.name = name; this[_privateData] = { age, password: 'secret123' }; } getAge() { return this[_privateData].age; } } const user = new User('John', 25); console.log(user.name); // 'John' (public) console.log(user[_privateData]); // { age: 25, password: 'secret123' } console.log(Object.keys(user)); // ['name'] (private data hidden!)

2. Avoiding Property Name Conflicts

Library Integration
// Library A uses symbols to avoid conflicts const LibraryA_ID = Symbol('LibraryA_ID'); const LibraryB_ID = Symbol('LibraryB_ID'); const myObject = { name: 'Test Object', [LibraryA_ID]: 'A123', [LibraryB_ID]: 'B456' }; // No conflicts even if both libraries use 'ID' console.log(myObject[LibraryA_ID]); // 'A123' console.log(myObject[LibraryB_ID]); // 'B456'

3. Well-Known Symbols (Built-in)

Custom Iterator
const myIterable = { data: ['a', 'b', 'c'], [Symbol.iterator]() { let index = 0; return { next: () => { if (index < this.data.length) { return { value: this.data[index++], done: false }; } return { done: true }; } }; } }; // Now it's iterable! for (const item of myIterable) { console.log(item); // 'a', 'b', 'c' }

🔧 Symbol Properties & Methods

Method/Property Description Example
Symbol(description) Creates a new symbol with optional description Symbol('mySymbol')
Symbol.for(key) Gets/creates a global symbol Symbol.for('global')
Symbol.keyFor(symbol) Gets the key of a global symbol Symbol.keyFor(sym)
symbol.description Gets the description of a symbol sym.description
symbol.toString() Returns string representation 'Symbol(desc)'
Symbol Methods Example
// Creating symbols const localSym = Symbol('local'); const globalSym = Symbol.for('global'); // Properties console.log(localSym.description); // 'local' console.log(localSym.toString()); // 'Symbol(local)' // Global symbol registry console.log(Symbol.for('global') === globalSym); // true console.log(Symbol.keyFor(globalSym)); // 'global' console.log(Symbol.keyFor(localSym)); // undefined

⚖️ Symbol vs Other Data Types

Feature Symbol String Number
Uniqueness ✅ Always unique ❌ Can be identical ❌ Can be identical
As Object Key ✅ Yes, hidden from enumeration ✅ Yes, visible ✅ Yes, converted to string
JSON.stringify ❌ Ignored ✅ Included ✅ Included
for...in loop ❌ Not enumerated ✅ Enumerated ✅ Enumerated
Object.keys() ❌ Not included ✅ Included ✅ Included
Comparison Example
const sym = Symbol('test'); const obj = { name: 'John', age: 30, [sym]: 'hidden value' }; console.log(Object.keys(obj)); // ['name', 'age'] console.log(Object.getOwnPropertySymbols(obj)); // [Symbol(test)] console.log(JSON.stringify(obj)); // {"name":"John","age":30} // Symbol property is there, but hidden! console.log(obj[sym]); // 'hidden value'

⚠️ Important Notes

  • Cannot use `new`: new Symbol() throws TypeError
  • Not coercible: Cannot be converted to string/number automatically
  • Property enumeration: Symbol properties are not enumerated by default
  • JSON serialization: Symbol properties are ignored in JSON.stringify()
